Since we all have Github. We can all start with Git Bash (command line interface).

The link to installation pack is here. As long as Git is in your windows environment path, you can use git commands in Visual Studio aswell. :)



1. Always create a repository using github website. Git Bash and Github is different. 

2.You cannot create a local repository using Git Bash and push that remotely to Github to create a new repository.

3.However, you can create a .git project locally and push to a remote repository on Github.



Scenario 1: You have a repository on Github (remote) created by you or someone else that you would like to contribute or copy the file to your local machine.

Step 1. Go to the repo and copy the https clone url.

Step 2. Open git cli/VS terminal/command prompt, navigate to the place you want to place the files and clone using: 

git clone <clone url> [filename]



Scenario 2: You created a new repository(remote) and would like to push all the local files to a repository on Github(remote).

Step 1. Navigate to the file, and initialise a .git project using:

git init

Step 2. Copy the clone url of your new repo from github.(similar to scenario 1, step 1)

Step 3. Set up a remote url using Git Bash by using:

git remote add origin <clone url>

so your files will be pushed to the correct repository(remote) on Github.

Step 4. Check you have the correct remote url setup by using:

git remote -v  

Example:

origin  https://github.com/winstoncwang/PicSearch.git (fetch)

origin  https://github.com/winstoncwang/PicSearch.git (push)

Step 5. Write some code and push the update with commits:

git status - checks for any unstaged update when you write new code.

git add <filename> / git add . - adds individual file or all the files before writing commits.

git commit -m"<title>" -m"<content>" - adds commits to your codes before pushing to remote.

git push <branch name>- push all your codes to the remote repository on github.


Most used commands:

git status - 

check the new code/file, give important information on unstage and staged files(staged files will be pushed to remote repo after commit).

git log - 

see all the previous commits, excellent way to check what you did and reset/revert if something is wrong.

git branch - 

a default master branch is initiated at repo creation, addtional branches are created when development is happening which later get merged with master. This is to prevent any bugs or error at deployment. Hence have different versions( version control ). 

git checkout <branch name> - 

this lets user work on different branches, possibility of working on different versions at the same time. Yeay

git checkout -b <branch name> -

creates a new branch and checkout on the branch

git reset <commit sha1 reference> -

undo the commit/code pushed.
